# RSS订阅文章评估与总结 - 提示词模板

## 版本信息
- **版本**: v1.0
- **创建日期**: 2025-12-31
- **适用场景**: RSS订阅源的自动化分析与高质量内容筛选

---

## 核心提示词（简化版）

```
分析 rss.opml 订阅文件，完成以下任务：

1. 解析所有 RSS 源，获取最新文章（每个源不用限制数量）
2. 对每篇文章进行质量评分（1-10分）
3. 筛选高分文章（≥7分）并生成详细报告
4. 输出到按日期命名的 Markdown 文件（格式：rss-summary-YYYY-MM-DD.md）

评分标准：
- 技术深度（30%）
- 实用价值（30%）
- 内容质量（20%）
- 时效性（20%）

报告需包含：
- 文章标题、来源、链接、发布日期
- 评分及推荐理由
- 核心主题分析
- 分类阅读建议
- 保留所有原文链接
```

---

## 详细提示词（完整版）

```markdown
# 任务：RSS订阅文章智能评估与总结

## 输入文件
- 文件路径：rss.opml
- 文件格式：OPML (XML格式的RSS订阅列表)

## 执行步骤

### 步骤1：解析订阅源
1. 读取 rss.opml 文件，提取所有 RSS feed URLs
2. 识别订阅源分类（如：技术博客、行业资讯、开发工具等）
3. 使用 RSS reader skill 批量解析订阅源
4. 每个源获取最近10-15篇文章

### 步骤2：文章质量评估
对每篇文章进行10分制评分，评估维度：

**技术深度 (30%)**
- 9-10分：原创性强，技术洞察深刻，有代码/架构分析
- 7-8分：有一定技术含量，案例详实
- 5-6分：技术介绍较浅，缺乏深度
- 1-4分：纯资讯或营销内容

**实用价值 (30%)**
- 9-10分：可直接应用，有完整示例和最佳实践
- 7-8分：提供有价值的思路和方法
- 5-6分：有一定参考价值
- 1-4分：价值有限

**内容质量 (20%)**
- 9-10分：逻辑清晰，论证充分，有数据支撑
- 7-8分：结构完整，表达清楚
- 5-6分：基本可读
- 1-4分：质量较差

**时效性 (20%)**
- 9-10分：最新技术/热点话题
- 7-8分：近期发布，仍有参考价值
- 5-6分：内容稍显陈旧
- 1-4分：过时内容

### 步骤3：内容分类
将高分文章（≥7分）分为：
- 🌟 **顶级文章** (9-10分)
- 🔥 **优秀文章** (7-8分)
- 💡 **值得关注** (6-7分)

### 步骤4：主题提取
识别文章的核心主题，常见主题包括：
- AI/机器学习
- 编程语言与框架
- 开发工具与效率
- 软件架构
- 行业动态
- 开源项目
- 安全与隐私
- 其他

### 步骤5：生成报告
输出到文件：`rss-summary-{今日日期}.md`

**报告结构**：

```
# RSS订阅文章评估报告
**生成日期**: YYYY-MM-DD

## 📊 高质量文章汇总

### 🌟 顶级文章 (评分 9-10/10)
（每篇包含）
- 标题
- 来源
- 链接（完整URL）
- 发布日期
- 评分
- 推荐理由（3-5句话）
- 核心内容摘要

### 🔥 优秀文章 (评分 7-8/10)
（同上格式）

### 💡 值得关注 (评分 6-7/10)
（同上格式）

## 📈 核心主题分析
（按主题归类分析）
- 主题1：简述 + 关键文章列表
- 主题2：简述 + 关键文章列表
- ...

## 🎯 阅读建议

### 最推荐阅读的TOP 3
（详细说明推荐理由）

### 不同读者群体推荐
- 程序员/开发者：...
- 技术管理者：...
- AI从业者：...
- 工具爱好者：...

## 📝 统计数据
- RSS源数量
- 文章总数
- 高质量文章数
- 各评分区间分布

## 🔗 快速导航
- 按来源分类
- 按主题分类
```

## 输出要求

1. **文件格式**：Markdown
2. **文件命名**：rss-summary-YYYY-MM-DD.md（使用当天日期）
3. **链接完整性**：确保所有原文链接可点击
4. **可读性**：使用emoji、分级标题、列表提升可读性
5. **客观性**：评分基于标准，避免主观偏见

## 特殊要求

- 优先解析以下高质量源（如果存在）：
  - Martin Fowler
  - 阮一峰的网络日志
  - 美团技术团队
  - ThoughtWorks洞见

- 自动跳过：
  - 解析失败的源
  - 纯广告/营销内容
  - 重复文章

- 处理异常：
  - RSS源访问失败：记录但继续处理其他源
  - 编码问题：尝试UTF-8解码
  - 超时：设置合理超时时间（30秒/源）

## 提示词优化建议

### 如需更详细的评估
添加以下维度：
- 作者权威性
- 引用来源可靠性
- 社区反馈（点赞/评论）
- 代码质量（如有）

### 如需自定义评分权重
修改评分公式：
```
总分 = 技术深度×0.3 + 实用价值×0.3 + 内容质量×0.2 + 时效性×0.2
```

### 如需过滤特定主题
添加主题过滤器：
```
只保留以下主题的文章：
- AI/机器学习
- 云原生
- 前端开发
```

### 如需多语言支持
指定语言偏好：
```
优先级：中文 > 英文 > 其他
对英文文章提供简要中文总结
```
